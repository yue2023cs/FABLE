# FABLE: A Localized, Targeted Adversarial Attack on Weather Forecasting Models

**Authors**: Yue Deng, Asadullah Hill Galib, Xin Lan, Jack Gunn, Pang-Ning Tan, Lifeng Luo

**Paper Link**: https://arxiv.org/abs/2505.12167

# Abstract

Deep learning-based weather forecasting (DLWF) models have recently demonstrated significant performance gains over gold-standard physics-based simulation tools. However, these models are potentially vulnerable to adversarial attacks, which raises concerns about their trustworthiness. In this paper, we investigate the feasibility and challenges of applying existing adversarial attack methods to DLWF models and propose a novel framework called FABLE (Forecast Alteration By Localized targeted advErsarial attack) to address them. FABLE performs a 3D discrete wavelet decomposition to disentangle the spatial and temporal components of the data. By regulating the magnitude of adversarial perturbations across different components, FABLE produces adversarial inputs that remain closely aligned with the original inputs while steering the DLWF models toward generating the targeted forecast outcomes. Experimental results on real-world weather datasets demonstrate the effectiveness of FABLE over baseline methods across various metrics.

For an attack to be successful, the adversarial inputs for DLWF models should satisfy the following criteria:

1. *faithfulness*: the adversarial inputs should produce the intended forecasts at the targeted locations while causing minimal changes at non-targeted locations;
2. *closeness*: the adversarial inputs should remain close to the original inputs to avoid detection;
3. *geospatio-temporal realisticness*: the adversarial inputs should preserve the dependencies within the original inputs to enhance stealthiness.

However, as shown in this work, balancing these criteria is challenging for existing adversarial attack methods. For example, although TAAOWPF successfully alters the forecast at the targeted locations in the Figure, the generated adversarial input still contains noticeable perturbations in non-targeted regions, making the attack easier to detect. These added perturbations also alter the inherent spatial autocorrelations of the data.

**Our goal is to achieve adversarial attacks on DLWF models that satisfy faithfulness, closeness, and geospatio-temporal realisticness simultaneously.**

## How FABLE Works

**Proposed framework**, consisting of:

1. a **decomposition** step to extract 3D wavelet coefficients $`\mathbf{C}`$ from the original input $`\mathbf{X}`$
2. a **perturbation** step to generate $`\mathbf{C}'`$ that minimizes the total loss $`\mathcal{L}`$
3. a **reconstruction** step using the inverse 3D wavelet transform to generate the adversarial input $`\mathbf{X}'`$

In particular, we propose a **perturbation strategy** over different sub-bands to achieve our goal, and provide both empirical and theoretical support in the paper.

# run_example.ipynb Overview

The file `run_example.ipynb` provides a complete implementation of multiple **adversarial attack** methods against the **CLCRN** model on the **NLDAS precipitation** dataset.

The currently supported attack methods include:

- **FGSM**
- **TAAOWPF**
- **Noise Attack**
- **STPGD**
- **ALA**
- **FABLE**

## Notebook Structure

`run_example.ipynb` mainly consists of the following sections:

### 1. Cell Config

This section is used for basic environment and experiment setup, including:

- importing the required Python packages
- setting hyperparameters
- loading the dataset
- recording key statistical information used in the experiments

### 2. Cell Import CLCRN Model

This section loads the pre-trained **CLCRN** model for subsequent adversarial attack experiments.

### 3. Cell Import Adversarial Attack Methods

This section contains the class definitions or imports for different adversarial attack methods, which are used to instantiate specific attackers later.

### 4. Cell Implement Adversarial Attack

This section is the main driver of `run_example.ipynb`, and executes the full adversarial attack pipeline, including:

- initializing the attack method
- generating adversarial examples
- running the attack process

### 5. Cell Implement Evaluation

This section evaluates the performance of the adversarial attack results and analyzes how the attacks affect model performance.

### 6. Cell Construct Adversarial Target

This section constructs the **adversarial target** required for the attack process and provides the target definition for subsequent attacks.

## Functionality

By running `run_example.ipynb`, users can:

- reproduce multiple adversarial attack methods on the NLDAS precipitation dataset
- test attacks against a pre-trained CLCRN model
- evaluate the effectiveness of different attack methods
- analyze the robustness of the model under adversarial perturbations
